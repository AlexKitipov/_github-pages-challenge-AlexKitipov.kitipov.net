Demo template for personal site setup
Това репо е невалиден демострационен шаблон. Целта му е да покажe структура и файлове за GitHub Pages сайт. Не е предназначено за реална употреба с моите данни.

What this is
Purpose: Учебен пример как да си стартираш личен сайт с GitHub Pages.

Status: Невалиден шаблон за демонстрация — заменете всички мои данни с ваши.

Source copy: Тази страница е копие на: https://github.com/AlexKitipov/_github-pages-challenge-AlexKitipov.kitipov.net

What you must replace
Name and contact: Заменете „Aleksandar/AlexKitipov“, имейл и социални връзки с вашите.

Brand/domain: Премахнете или сменете всякакви препратки към kitipov.net, kitipov.cu.ma, kitipov.dev.

Assets: Подменете изображения, лога и аудио файлове с ваши.

Texts: Редактирайте index.html, about.html, README.md и метаданни (title, description).

Tip: Търсете по ключови думи „Kitipov“, „Algorithm of the Day“, и ги заменете навсякъде.

Step-by-step: Make your official site
Create the special repository

Име на репото: username.github.io заменете username с вашия GitHub потребител (пример: maria-ivanova.github.io).

Add a minimal homepage

Създайте файл index.html с вашето съдържание.

Можете да копирате структура от този шаблон и да замените данните.

Enable GitHub Pages

В Settings → Pages → Source: изберете branch main (или master) и root.

Изчакайте няколко минути — сайтът ще бъде достъпен на: https://username.github.io

Optional: Custom repository site

Ако искате втори сайт за проект: създайте отделно репо и активирайте Pages.

Ще се показва на: https://username.github.io/project-name

Optional: Custom domain

Ако по-късно добавите домейн, следвайте инструкциите за CNAME и TXT верификация.

Докато нямате домейн, официалният безплатен адрес остава https://username.github.io.

Notes and disclaimers

Privacy: Не публикувайте чужди лични данни. Заменяйте и премахвайте всичко, което не ви принадлежи.
